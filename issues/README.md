# 이슈 관리

이슈를 상태별로 관리하는 폴더입니다.

## 폴더 구조

```
issues/
├── .templates/       # 이슈 작성용 템플릿
│   ├── issue-template.md   # 기본 이슈 템플릿
│   ├── bug-template.md     # 버그 이슈
│   └── feature-template.md # 기능 추가 이슈
├── 진행예정/         # 계획은 세웠으나 아직 시작하지 않은 이슈
├── 진행중/           # 현재 작업 중인 이슈
├── 완료/             # 완료된 이슈
├── 일시정지/         # 의존성이 생겨 잠시 보류 중인 이슈
└── 폐기/             # 더 이상 진행하지 않는 이슈
```

## 이슈 유형 (`type`)

| 유형 | 설명 |
|------|------|
| 버그 | 작동하지 않거나 예상과 다른 동작 |
| 기능 | 새로 추가할 기능 |
| 개선 | 기존 기능의 성능/UX 개선 |
| 리팩터 | 코드 품질 개선(기능 변경 없음) |
| 문서 | README, GDD, 주석 등의 문서화 |
| 설정 | CI/CD, 빌드, 의존성 등 환경 설정 |

## 라이프사이클

```
진행예정 ──▶ 진행중 ──▶ 완료
  │            │
  │            ▼
  │          일시정지 ──▶ 진행중
  │
  ▼
  폐기
```

## 이름 규칙

이슈 파일명: `ISSUE-XXX_간단한설명.md`

- 번호는 중복되지 않게 부여합니다.
- 간단한 설명은 한글로 5~10단어 이내로 적습니다.

## 작성 방법

1. `.templates/`에서 유형에 맞는 템플릿을 복사합니다.
2. 이슈 ID와 내용을 채워 `진행예정/`에 넣습니다.
3. 작업을 시작할 때 `진행예정/`에서 `진행중/`으로 이동합니다.
4. 완료 시 `진행중/`에서 `완료/`으로 이동합니다.
